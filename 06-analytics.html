Назначение раздела «Аналитика»

Переход осуществляется через пиктограмму аналитики в левом rail-меню. После открытия пользователь попадает на вкладку «Генерация отчётов». KPI-карточки показывают текущее состояние КД без фильтров.

Формат A/B в ячейках таблицы

A — количество документов, фактически согласованных данным подразделением за отчётный период (нарастающим итогом с начала этапа).
B — количество документов, возвращённых этим подразделением на доработку. Возврат означает, что конструктор должен устранить замечания и повторно направить документ по маршруту.

Красная ячейкаПоказатель превышает нормативный порог контрольной даты. Требуется эскалация ответственному.
«На доработке»Документ возвращён согласующим — находится у конструктора для устранения замечаний.
«Пересогласование»После устранения замечаний документ вновь направлен по маршруту согласования.
Дата в заголовкеСостояние на начало рабочего дня указанной даты (срез БД в 08:00).

Гистограмма «Выпущено по КБ»

Показывает результативность каждого конструкторского бюро за выбранный период. Высота столбца пропорциональна количеству документов, прошедших все этапы согласования. КБ ИЦ традиционно лидирует по объёму — это головное КБ проекта МС-21.

Круговая диаграмма «Структура по типу»

Изменение извещения (ИИ) составляют наибольшую долю — это рабочий поток корректировки серийных чертежей. Технические регламенты (ТР) проходят удлинённый маршрут с согласованием Главного конструктора.

Гистограмма «Поступило / Согласовано»

Пары столбцов сравнивают входящую нагрузку с фактическим результатом. В апреле согласовано больше, чем поступило — КБ «отработало» накопленный задел предыдущих месяцев.

Интерпретация фильтра

Фильтр «КБ ИЦ» ограничивает выборку документами, выпущенными этим КБ. Фильтр «Технологический контроль» показывает только те документы, которые находятся или прошли данный этап маршрута. Сочетание фильтров позволяет руководителю КБ ИЦ видеть узкое место технологического контроля и сравнивать фактическое время с нормативным.
