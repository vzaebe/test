## С чего начать

1. **Откройте `index.html`** в браузере — это каталог всех разделов.
2. Для гайдлайнов и правил откройте **`design-system.html`** — это канонический источник правды.
3. Прочитайте раздел **«Философия и принципы»** — это 7 правил, на которых построено всё.
4. Перед любыми изменениями интерфейса — изучите справочник компонентов и токенов.
5. Если меняете экран — обязательно изучите раздел **«Focus-mode»** и правило **«Цвет = смысл»**.

## Как меняется оформление

> **Один источник CSS — все файлы обновляются автоматически.**

Все 9 файлов-разделов подключают `assets/asskd-design-system.css` через `<link rel="stylesheet">`.
Любое изменение токена или компонента в этом файле моментально применяется ко всем разделам — переписывать каждый файл не нужно.

```html
<!-- В каждом файле раздела — строка подключения общей дизайн-системы -->
<link rel="stylesheet" href="assets/asskd-design-system.css"/>
```

## Как пересобрать файлы из исходника

Если нужно перегенерировать файлы разделов из `asskd_unified_v4.html` (например, после правок исходника):

```powershell
powershell -ExecutionPolicy Bypass -File .\build-sections.ps1
```

Скрипт перечитывает диапазоны строк исходника, оборачивает каждый раздел в общий шаблон с навигацией и хлебной крошкой и записывает 9 HTML-файлов + `index.html`.

## Главные правила (TL;DR)

1. **Цвет = смысл.** Один цвет всегда означает одно состояние. См. справочник статусов.
2. **Только токены.** Все цвета, отступы, радиусы — через `var(--token)`. Никакого хардкода.
3. **Hover везде = `--blue-ll`.** Любой кликабельный элемент при наведении подсвечивается.
4. **Focus-mode для сложных экранов.** Если на экране 3+ панелей — применяйте `.has-focus` + `.is-focus`.
5. **Каждый экран — отдельный `<article class="scr">`** с уникальным ID `scr-{секция}-{номер}`.

## Структура репозитория

Монолитный файл `asskd_unified_v4.html` (4670 строк) разбит на отдельные документы по типам разделов руководства пользователя. Все они используют общую дизайн-систему через единый CSS-файл.

```
asskd/
├── index.html                         🧭 каталог разделов (точка входа)
├── design-system.html                 🟢 каноническая дизайн-система v1.0
├── assets/
│   └── asskd-design-system.css        🎨 единая таблица стилей
├── 01-design-system.html              📐 Дизайн-система + справочники
├── 02-entry-home.html                 🚪 Вход · Главная
├── 03-my-processes.html               📋 Мои процессы (13 экранов)
├── 04-document-issuance.html          📤 Выпуск документов (15 экранов)
├── 05-document-approval.html          ✅ Согласование документов (16 экранов)
├── 06-analytics.html                  📊 Аналитика (22 экрана)
├── 07-process-monitoring.html         👁  Наблюдение за процессами (17 экранов)
├── 08-logout.html                     ↩  Выход из системы
├── 09-statuses-reference.html         🗂  Справочник статусов (Прил. 3–5)
├── asskd_unified_v4.html              🟡 исходный монолитный концепт (заморожен)
├── build-sections.ps1                 🛠 генератор файлов разделов
└── README.md
```

Подробный план миграции и принципы — в разделах **«Философия и принципы»** и **«Структура проекта»** файла `design-system.html`.

## Для AI-ассистентов

Если работаете через Cursor / Copilot / другого AI-ассистента — копируйте этот промпт перед запросами:

```
При работе с проектом АССКД КД соблюдай:
1. Читай design-system.html перед любыми изменениями интерфейса.
2. Используй ТОЛЬКО токены из :root — никаких хардкод-цветов.
3. Цвет статуса = смысл (см. раздел «Справочник статусов»).
4. Hover для всех интерактивных = var(--blue-ll).
5. Каждый экран = <article class="scr" id="scr-N-NN">.
6. Если нужен новый компонент — сначала добавь его в design-system.html.
7. Для сложных экранов с 3+ панелями применяй .has-focus + .is-focus.
8. Не трогай файл asskd_unified_v4.html — он зафиксирован как concept-snapshot.
```
