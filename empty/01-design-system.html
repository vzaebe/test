Правила построения
Контейнер — всегда .panel с .panel-h (заголовок + подзаголовок) и .panel-b (тело). Без заголовка график не используется.
Высота — стандарт 200 px (.chb2), компакт 180 px. Поле баров — 115–120 px от низа, отступ bottom:26px под подписи.
Бары — flex:1, gap 8 px, радиус 4 4 0 0. Цвет = смысл (см. ниже).
Легенда обязательна. Класс .legend: i 8×8 px + текст 10/600 muted. У круговой — justify-content:center.
Круговая — 150×150 px, не более 4 секторов. Использовать conic-gradient в порядке: blue → sky → teal → gray.
Нельзя: 3D-эффекты, тени на барах, повороты круговых, более 4 цветов, цвета вне палитры брендбука.
Цвет = смысл (синхронизировано со статусами)

Правила колонок и строк
thead — высота 32 px, фон --n50, текст 10/700 uppercase muted, letter-spacing .04em.
tbody td — высота 32–36 px, padding 8×10/12, border-bottom --line3. Hover строки — --blue-ll.
Ключевое поле (номер, название) — <strong>, без uppercase. Класс .nw на колонках с датой/номером.
Статус — только через .bg .bi/.bs/.bd/... (бейдж + точка). Никогда — голым текстом или цветным td.
warncell — для критичных значений (просрочка, 0/N задач): color:var(--err), фон --err-ll.
Сортировка — стрелка ↑/↓ прямо в th. Активная — у одного th в таблице.
Нельзя: цветной фон ячейки без смысла, более 1 «жирного» поля в строке, иконки вместо бейджа статуса, вертикальные разделители колонок.

Общие правила применения
Один тип = один контекст. Маркированный — только в hero/callout, нумерованный — только для последовательных шагов, kv — только для атрибутов.
Gap 6–8 px между элементами. Padding элемента 8×10 px. Никаких дефолтных list-style, ul/ol markers.
